Terraform GCP Portfolio Website

By Chinonso Onyechi, Cloud Architect and Engineer

Project Overview

This project demonstrates how to deploy a static website via Google Cloud Storage using the Infrastructure as Code (IaC) tool Terraform, fully automated and managed as code. The inspiration behind this project is to provide a beginner-friendly example for those new to cloud computing and Terraform—a practical, approachable starting point to build confidence and skills.

Why This Project?

When I started my cloud journey, I didnt know where to begin. This project serves as the foundation for a cloud project playbook aimed at beginners—a resource I wish I had when starting out. It guides aspiring professionals through fundamental cloud concepts and Terraform practices, helping them build a real, deployable project from scratch.

Steps Taken

  1. Installed Terraform & Google Cloud SDK on my local machine.
  2. Authenticated with Google Cloud using gcloud auth login.
  3. Enabled the Google Cloud Storage API.
  4. Created Terraform configuration files (main.tf, variables.tf, outputs.tf).
  5. Configured a public Google Cloud Storage bucket with website hosting enabled.
  6. Defined IAM policies to allow public read access to the bucket contents.
  7. Uploaded static website files (HTML + CSS) as bucket objects using Terraform resources.
  8. Initialized and applied the Terraform plan to deploy all resources.
  9. Tested the live website via the Terraform output URL.
  10. Version controlled the entire project, including infrastructure and website source code, using GitHub for professional collaboration and portfolio presentation.
